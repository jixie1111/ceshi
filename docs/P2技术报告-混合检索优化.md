# P2 技术报告：面向中文医学教材的混合检索与引用命中率优化

## 摘要

本项目提出一套适合 5 小时黑客松落地的 RAG 检索优化方案：字符级 TF-IDF / sentence-transformers 向量检索、BM25 关键词检索与问题词 overlap rerank 融合。该方案的目标不是单纯提升语义召回，而是提升“回答引用是否来自正确教材章节”的可评测指标。系统内置 `/api/benchmark/run`，评审上传 7 本教材后可自动生成引用命中率、响应时间与 top citation 数据。

## 1. 问题分析

中文医学教材存在三个检索难点：

1. 专有名词密集：如“压力感受性反射”“动作电位”“肾小球滤过率”。
2. 同义表述多：如“白细胞 / leukocyte / white blood cell”。
3. 章节跨度大：一个问题可能需要从不同教材或不同章节抽取证据。

仅用向量检索时，语义相近但章节错误的 chunk 可能被召回；仅用 BM25 时，同义表述召回不足。

## 2. 方案设计

最终分数：

```text
score = 0.62 * vector_score + 0.28 * bm25_score + 0.10 * keyword_overlap
```

- vector_score：支持 sentence-transformers；不可用时回退到 char n-gram TF-IDF。
- bm25_score：中文分词后计算关键词稀疏匹配。
- keyword_overlap：问题词与 chunk 词交集比例，作为 rerank 的轻量补偿。

## 3. 实验设计

系统运行后，在“评测”Tab 点击“运行 RAG Benchmark”。自动生成每本教材前 4 个章节的问题：

```text
{章节标题} 的核心内容是什么？
```

指标：

- Citation Hit Rate：top-3 引用中是否包含预期章节。
- Latency：端到端检索与生成耗时。
- Top Citation：第一条引用的教材、章节和页码。

## 4. 工程洞察

- chunk 过短会切断定义和机制；chunk 过长会稀释检索分数。
- 中文医学教材的精确术语占比高，BM25 对引用正确性帮助明显。
- overlap 不需要太大，80 字足以覆盖定义跨段情况。
- 无模型环境下，char n-gram TF-IDF 对中文术语仍有可靠表现。

## 5. 局限与未来方向

- 当前 benchmark 以章节标题生成问题，覆盖事实性问题较好，跨教材推理题仍需人工标注。
- 后续可引入 bge-reranker 或 cross-encoder 做二阶段重排。
- 可把教师确认的正确引用作为弱监督数据，动态调整检索权重。
