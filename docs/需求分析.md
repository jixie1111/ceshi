# 需求分析

## 1. 问题拆解

赛题核心目标是把多本教材压缩为不超过原始体量 30% 的知识精华，同时保留教学效果与可追溯证据链。问题被拆为 6 个子问题：

1. 多格式教材解析：PDF / Markdown / TXT 必须支持，DOCX / Excel 作为增强。
2. 单书知识图谱：从章节抽取概念、方法、机制、现象等核心知识点，并识别关系。
3. 跨教材语义对齐：判断不同教材中的同义、近义、互补知识点。
4. 压缩与教学完整性：合并重复内容，保留关键链路，输出 ≤30% 的精华内容。
5. RAG 精准问答：每个回答必须有教材、章节、页码与原文 chunk。
6. 教师多轮反馈：教师可通过自然语言修改整合决策，避免算法误删关键知识点。

## 2. 知识点粒度定义

知识点粒度选择“章节内可独立讲授的最小概念/机制/指标/方法”。例如：

- 概念：静息电位、动作电位、心输出量、内环境稳态。
- 机制：压力感受性反射、钠通道开放导致去极化。
- 方法/指标：心电图、基础代谢率、清除率。
- 应用场景：临床诊断、治疗靶点、功能评价。

该粒度比段落小、比关键词大，适合图谱关系建模与 RAG 引用。

## 3. 重复判定标准

重复不是简单字符串相等，而是“教学语义等价”：

1. 名称归一化后相同：如 leukocyte / 白细胞。
2. 名称高相似：RapidFuzz 相似度 ≥ 0.93。
3. 跨书语义相似：向量相似度 ≥ 0.82。
4. 双重判定：语义相似 ≥ 0.76 且名称相似 ≥ 0.72。

同一本教材内部更保守，避免误把上下位概念合并；跨教材更积极，满足去重提纯目标。

## 4. 压缩比计算

```text
压缩比 = 整合后精华字数 / 原始教材总字数 × 100%
```

整合后精华字数由合并节点定义、关系说明、教师锁定保留内容构成。系统显示：

- 原始总字数
- 整合后字数
- 压缩比百分比
- 节点数和关系数变化

若精华字数超过 30%，报告中会优先保留高频节点、前置依赖节点、教师锁定节点，并压缩低重要度定义。

## 5. 教学连贯性保障

压缩不等于简单删除。系统保留以下教学结构：

1. prerequisite：保证先修概念不丢失。
2. contains：保证章节主题与子概念覆盖完整。
3. parallel：保留对比学习材料。
4. applies_to：保留基础与临床/应用场景的桥梁。
5. teacher_locked：教师反馈锁定关键知识点，优先级高于算法压缩。

## 6. RAG 分块策略

分块大小选 **500–800 字**，默认 700 字，overlap 80 字。

理由：

- 中文医学教材定义与机制描述通常跨 1–3 个自然段，700 字能容纳完整语义。
- 80 字 overlap 可减少定义被截断风险。
- top-5 检索上下文约 3500 字，适合注入 LLM，也适合无 LLM 的抽取式回答。
- Benchmark 可对比 300 / 700 / 1000 字，当前默认在准确率与响应时间之间折中。
